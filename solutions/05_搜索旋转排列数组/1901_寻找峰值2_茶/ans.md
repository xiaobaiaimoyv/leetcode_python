    ## 核心思路：行二分 + 行最大值
  
    这个算法的精髓在于：**用二分法锁定“行”，用贪心法锁定“列”。**
    
    ### 1. 为什么二分“行”？
    
    如果我们一行一行找，那是 $O(m \times n)$。为了降维，我们对行坐标 $i$ 进行二分（`top` 和 `bottom`）。每次我们只看中间那一行 `mid`。
    
    ### 2. 为什么在 `mid` 行找“最大值”？
    
    这是为了**消除左右方向的干扰**。
  
    - 如果 $mat[mid][j]$ 是第 `mid` 行的最大值，那么它一定满足：
    
      $$mat[mid][j] \ge mat[mid][j-1] \quad \text{且} \quad mat[mid][j] \ge mat[mid][j+1]$$
    
    - 此时，这个点是否是“二维峰值”，**只取决于它的上下邻居**。
    
    ------
    
    ## 代码逐行拆解
    
    Python
    
    ```
    class Solution:
        def findPeakGrid(self, mat: List[List[int]]) -> List[int]:
            top = 0
            bottom = len(mat) - 1
            
            while top <= bottom:
                # 1. 确定搜索的行范围，取中间行
                mid = (top + bottom) // 2
                
                # 2. 在当前行寻找“绝对最大值”的列索引 max_col
                # 这一步保证了：该点已经打败了左右邻居
                max_col = 0
                for j in range(len(mat[mid])):
                    if mat[mid][j] > mat[mid][max_col]:
                        max_col = j
                
                # 3. 纵向对比：检查上下邻居
                # 使用逻辑判断处理边界（如果是在第一行或最后一行，视作邻居无限小）
                is_greater_than_top = (mid == 0) or (mat[mid][max_col] > mat[mid-1][max_col])
                is_greater_than_bottom = (mid == len(mat)-1) or (mat[mid][max_col] > mat[mid+1][max_col])
                
                # 4. 判断结果或缩小区间
                if is_greater_than_top and is_greater_than_bottom:
                    # 四面皆小，这就是我们要找的峰值
                    return [mid, max_col]
                
                elif not is_greater_than_top:
                    # 核心逻辑：如果上方邻居更大，说明“更高处”在上面
                    # 根据“爬山原理”，上半区一定存在至少一个峰值
                    bottom = mid - 1
                else:
                    # 反之，下半区一定有答案
                    top = mid + 1
                    
            return [-1, -1]
    ```
    
    ------
    
    ## 算法原理的直观理解
    
    ### 爬山法则 (The Climbing Rule)
    
    想象你在一个黑夜里的山区，只能看到脚下的路和前、后、左、右一步的距离：
    
    1. **横向移动**：你先在当前的纬度（行）走到最高点。
    2. **纵向观察**：你看看北边（上）和南边（下）。
    3. **决策**：
       - 如果南北都比你低，你就在山顶了！
       - 如果南边更高，你就往南边跨一步。
       - **关键点**：当你跨到南边那一行后，你再次寻找那一行（新纬度）的最高点。因为你刚才跨过来时已经变高了，而新一行的最高点只会让你**变得更高或保持高度**。
    
    ### 为什么一定能找到？
    
    因为矩阵的数值是**有限的**。由于你每一步都在往“更高”的地方走（或者在二分缩小的区域里找更高），你不可能永远走下去。最终，你会被逼到一个“四面环低”的点，那就是我们要找的峰值。
    
    ------
    
    ## 复杂度分析
    
    - **时间复杂度**：$O(n \log m)$
      - 二分行需要 $\log m$ 次迭代。
      - 每次迭代中，遍历列寻找最大值需要 $O(n)$。
    - **空间复杂度**：$O(1)$
      - 只使用了常数个变量，非常节省内存。
